Bot Setup Guide

Build a personal AI that runs 24/7 on your own hardware. Each phase unlocks a new capability. Check off steps as you go.

Setup
Get a dedicated machine running with OpenClaw and remote access. After this, the AI is alive but you haven't talked to it yet.
Get a dedicated machine

Your AI needs its own machine that never turns off. Two good options:

Mac mini or spare laptop (recommended): Small, silent, low power ($2-3/month electricity). A Mac mini is ideal. A spare laptop with the lid closed also works. This should not be the computer you use every day.

VPS (cloud server): Rent one from Hetzner or DigitalOcean for $5-10/month. Good if you don't have spare hardware or want it running in a datacenter. SSH access required.

Open a terminal

Mac mini / laptop: Press Command (⌘) + Space to open Spotlight. Type terminal and press Enter. A window with a blinking cursor opens.

VPS: From your personal computer, open Terminal and connect via SSH:

copyssh root@your-vps-ip
Install prerequisites

Mac: Install Homebrew (a package manager). Copy this line, paste into Terminal, press Enter:

copy/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

It asks for your Mac password. When you type it, nothing appears on screen. That's normal. Type it and press Enter. Wait 2-5 minutes.

VPS (Ubuntu/Debian): Update packages:

copyapt update && apt upgrade -y
Prevent sleeping (Mac only)

Your AI can't work if the machine sleeps:

copysudo pmset -a sleep 0

Also: System Settings → Displays → Advanced → turn off "Automatically adjust brightness." For laptops: Battery → Options → "Prevent automatic sleeping on power adapter."

VPS users: Skip this step. Cloud servers don't sleep.
Create a Gmail account for the AI

Your AI needs its own email for signing up to services. Go to Gmail signup on your phone (easier for phone verification). Pick any name.

Never use your personal Gmail. The AI's accounts are completely separate. If something goes wrong, only a throwaway email is at risk.
Install OpenClaw
copycurl -fsSL https://openclaw.ai/install.sh | bash

This installs OpenClaw, registers it as a background service (auto-starts on boot), and starts it immediately.

Mac alternative: brew install openclaw then openclaw gateway install && openclaw gateway start.

Install Tailscale

Tailscale (free) creates a private encrypted network between all your devices, no matter where they are. You'll use it for remote access and file syncing.

Mac:

copybrew install --cask tailscale

Open Tailscale from Applications. Click the menu bar icon and sign in. Note the Tailscale IP (like 100.x.x.x).

VPS (Linux):

copycurl -fsSL https://tailscale.com/install.sh | sh sudo tailscale up

Follow the auth URL it prints. Note the Tailscale IP.

Then install Tailscale on your phone (App Store / Play Store) and laptop (tailscale.com/download). Sign in with the same account on each.

Enable the OpenClaw dashboard
copyopenclaw config set gateway.controlUI.enabled true openclaw gateway restart

From any device on your Tailscale network, open http://[Tailscale IP]:18789. Bookmark it.

Enable screen sharing (Mac only)

System Settings → General → Sharing → turn on Screen Sharing. Access from another Mac: Finder → Go → Connect to Server → vnc://[Tailscale IP]. From phone: use a VNC app. Useful for CAPTCHAs and browser issues.

VPS users: You already have SSH access. No screen sharing needed since there's no GUI.
The machine is ready. OpenClaw is running, Tailscale gives you remote access, and the dashboard lets you monitor everything. Next: connect a messaging app so you can talk to it.
Chat
Connect a messaging app and start talking to your AI. After this, you have a working assistant.
Connect Telegram

Telegram is a free messaging app. This is how you'll talk to your AI -- like texting a friend who happens to be an AI.

Install it on your phone from the App Store or Play Store. Then follow the OpenClaw Telegram setup guide (~5 minutes -- you create a "bot" in Telegram and paste a token into OpenClaw).

Other options: OpenClaw also works with Signal, Discord, Slack, WhatsApp, and iMessage. Telegram is the easiest to start with.
Send your first message

Open Telegram and message your AI: "Hello, are you there?"

It should reply within seconds. If not, check the dashboard for errors. Common fix: openclaw gateway restart.

Give it a real task right away. Don't spend a week configuring. Ask something real: "Build me a personal website." "Research hotels in Kyoto for April." "Summarize this PDF." You learn by using it, not by tweaking settings.
Let the AI create its own accounts

Tell the AI to sign up for developer services:

"Sign up for GitHub, Vercel, and Supabase using your Gmail account."

It opens a browser, goes to each site, and handles registration. Help with CAPTCHAs via screen sharing (Mac) or paste verification codes (VPS).

For small purchases (like a $11/year domain), create a free virtual card at Privacy.com with a $20/month limit.

Share your existing projects

Invite the AI to collaborate:

GitHub: Repository → Settings → Collaborators → Add the AI's username.

Vercel: Team → Members → Invite the AI's Gmail.

Supabase: Organization → Members → Invite the AI's Gmail.

Google Docs: Share → Enter the AI's Gmail.

You now have a fully working AI assistant. It can write code, deploy websites, manage databases, search the web, and research anything. Everything after this makes the experience better -- none of it is required.
Your bot can finish the rest. Send your bot this message: "Read botsetupguide.com and set up Sync and Security for me. Ask me questions if you need input." The remaining phases are written so your AI can read and execute them directly. You can also continue manually below.
Sync
See everything your AI writes on all your devices. Two paths depending on your setup.
Which sync method? Choose based on your situation:

Already use Obsidian Sync? Keep using it. It works great and syncs plugin settings too. Follow "Path A" below.

Starting fresh, or on a VPS? Use Syncthing (free, peer-to-peer, trustless). Follow "Path B" below. This is the default recommendation.

VPS users: You must use Path B. Obsidian Sync requires the desktop app running with a GUI, which a headless VPS can't do.
Path A
Obsidian Sync (if you already have it)
You already pay for Obsidian Sync ($4/month). Keep using it. Skip Path B.
Path A: Install Obsidian on the AI's Mac

Obsidian makes Markdown files (your AI's native format) beautiful. On the AI's machine:

copybrew install --cask obsidian

Open Obsidian. Click "Open folder as vault." Press Command + Shift + G, type ~/.openclaw/workspace, click Open.

Path A: Enable Obsidian Sync

In Obsidian: gear icon → Core plugins → toggle on Sync → click Sync → log in → Create new remote vault.

In Sync settings, add Code to "Excluded folders". Code repos use Git, not Sync.

Leave "Sync all file types" off. Default (Markdown only) is correct. Code projects have thousands of build files that would overwhelm storage.
Path A: Keep Obsidian running forever

Obsidian only syncs while open. Auto-launch and self-restart:

copycat > ~/Library/LaunchAgents/md.obsidian.keepalive.plist << 'PLIST' Labelmd.obsidian.keepalive ProgramArguments open-aObsidian RunAtLoad StartInterval300 PLIST launchctl load ~/Library/LaunchAgents/md.obsidian.keepalive.plist
Path A: Connect your phone and laptop

Install Obsidian on your phone (App Store / Play Store) and laptop (obsidian.md/download).

On each device: new vault → Settings → Sync → same account → select the remote vault. Everything the AI writes appears on your phone within seconds.

Bonus: Obsidian Sync also syncs plugin data. Install the OpenClaw plugin on one device and it appears on all of them -- including the encryption keys. Pair once, works everywhere.
Path B — Recommended
Syncthing over Tailscale (free, trustless)
Peer-to-peer file sync. No server, no subscription, no one sees your data. Works with Mac mini, laptop, and VPS. Tailscale (from Setup phase) handles the networking.
Path B: Install Syncthing on the AI's machine

Mac:

copybrew install syncthing brew services start syncthing

VPS (Ubuntu/Debian):

copyapt install -y syncthing # Create a systemd service to run Syncthing as root (or your user) cat > /etc/systemd/system/syncthing.service << 'EOF' [Unit] Description=Syncthing After=network.target [Service] ExecStart=/usr/bin/syncthing serve --no-browser --no-restart Restart=on-failure User=root [Install] WantedBy=multi-user.target EOF systemctl enable syncthing && systemctl start syncthing

Syncthing's web UI runs at http://127.0.0.1:8384. Access it via Tailscale: http://[Tailscale IP]:8384.

VPS security: By default Syncthing only listens on localhost. If you access the web UI over Tailscale, set a username and password in Settings → GUI.
Path B: Install Syncthing on your devices

Mac/PC: syncthing.net/downloads or brew install syncthing.

iPhone: Mobius Sync (App Store, $6 one-time). The only iOS Syncthing client.

Android: Syncthing (free on Play Store).

Path B: Connect the devices

On the AI's machine, open Syncthing web UI (http://[Tailscale IP]:8384). Note the Device ID shown under Actions → Show ID.

On your device's Syncthing: Add Remote Device → paste the AI machine's Device ID. Syncthing will find it over Tailscale automatically.

Accept the pairing request on both sides.

Path B: Share the workspace folder

On the AI's machine Syncthing UI: Add Folder:

Folder Path: ~/.openclaw/workspace (Mac) or /root/.openclaw/workspace (VPS)

Folder Label: workspace

Under the Sharing tab, check the box for your device.

On your device, accept the folder share. Set the local path to wherever you want the files (your Obsidian vault location works great).

Ignore patterns: In the folder settings, add these to Ignore Patterns to skip code repos and build artifacts:
Code
.obsidian
node_modules
Path B: Install Obsidian and open the synced folder

Install Obsidian (free) on your phone and laptop. Open the synced workspace folder as a vault. Your AI's notes, memories, and files appear beautifully rendered -- and update in real time.

Both Paths
Install the OpenClaw chat plugin
Chat with your AI directly inside Obsidian. Works the same regardless of which sync path you chose.
Install the OpenClaw plugin

In Obsidian: Settings → Community Plugins → Browse → search "OpenClaw" → Install → Enable.

If the plugin isn't in the directory yet, install manually:

copy# From the AI's machine (or any machine with git): cd ~/.openclaw/workspace/.obsidian/plugins mkdir -p openclaw && cd openclaw curl -L -o main.js https://github.com/oscarhenrycollins/obsidianclaw/releases/latest/download/main.js curl -L -o manifest.json https://github.com/oscarhenrycollins/obsidianclaw/releases/latest/download/manifest.json curl -L -o styles.css https://github.com/oscarhenrycollins/obsidianclaw/releases/latest/download/styles.css

Then in Obsidian: Settings → Community Plugins → reload → enable "OpenClaw".

Connect the plugin to your gateway

The setup wizard appears on first launch. Enter your gateway URL:

Same machine: ws://127.0.0.1:18789

Over Tailscale: ws://[Tailscale IP]:18789

First connection triggers device pairing. On the AI's machine:

copyopenclaw devices list openclaw devices approve <requestId>
Three layers of security. Tailscale encrypts the network. The auth token verifies your identity. Ed25519 device fingerprinting prevents unauthorized devices. Your private key never leaves your machine.
Enable semantic search

Search all notes by meaning, not keywords. "What restaurant did James recommend?" works even if "restaurant" never appears. Runs locally -- no data sent anywhere.

copyopenclaw config set agents.defaults.memorySearch.provider local openclaw config set agents.defaults.memorySearch.fallback none

Tell it which folders to search. Open the config:

copyopen ~/.openclaw/openclaw.json

Find memorySearch and add your folders:

"extraPaths": ["Contacts", "Lists", "Memos", "Tasks"]

Save, close, restart:

copyopenclaw gateway restart

First run downloads a model (~313MB) and indexes notes (2-5 min). After that, instant.

Put all code in Code/. Every repo, project, and skill. Excluded from syncing (both Obsidian Sync and Syncthing). Code uses Git, notes use sync. Tell the AI: "Always clone into Code/."
Create a Whiteboard.md at the vault root. Shared scratchpad: paste links, jot thoughts, dump ideas. Say "check my whiteboard" and the AI knows what you mean.
Harden
You've given an AI real tools. Lock it down so the risk stays manageable.
Dedicated machine, separate accounts

The AI should never run on your personal computer or be logged into your personal accounts. Its own machine, Gmail, GitHub, everything. If compromised, the damage is a throwaway Gmail and free-tier accounts.

Gateway stays local

The OpenClaw gateway only listens on 127.0.0.1 (localhost). No one on the internet can reach it. Never expose it publicly. Tailscale handles remote access through a private encrypted tunnel.

VPS users: Make sure port 18789 is NOT open in your firewall. Tailscale bypasses the firewall safely. Check with: ss -tlnp | grep 18789 -- it should show 127.0.0.1, not 0.0.0.0.
Write safety rules

Open SOUL.md in your AI's workspace and add rules:

"Never send emails, make purchases, or post publicly without asking me first. Never access files outside the workspace. Use trash instead of permanent delete."

Like setting expectations with a new employee. Not unbreakable, but significantly reduces unintended actions.

Use scoped tokens with expiration dates

API tokens should have minimum permissions and expire after 90 days. Ask your AI: "Review your API tokens and flag any with excessive permissions or no expiration."

Enable the firewall (VPS)

If on a VPS, lock down everything except SSH and Tailscale:

copyufw default deny incoming ufw default allow outgoing ufw allow ssh ufw allow in on tailscale0 ufw enable

This ensures only Tailscale traffic reaches OpenClaw. Everything else is blocked.

Mac users: macOS firewall is off by default but Tailscale handles security. Enable it in System Settings → Network → Firewall if you want an extra layer.
Keep OpenClaw updated
copyopenclaw update

Run weekly, or tell the AI: "Run openclaw update."

What's still risky: Prompt injection (malicious content tricking the AI) is unsolved industry-wide -- that's why throwaway accounts matter. The LLM provider sees your conversations, so don't share passwords or financial data. Only install OpenClaw skills from trusted sources.
