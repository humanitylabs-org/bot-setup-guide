Bot Setup Guide

Build a personal AI assistant that runs 24/7 on your own machine. It reads your notes, writes code, deploys sites, and replies to your messages. Check off each step as you go.

Part 1 — The Machine
Get a computer that stays on 24/7

Your AI needs a permanent home. Best: Mac mini. Also works: spare laptop, Linux desktop, or cloud server (VPS). It must never fully sleep.

sudo pmset -a sleep 0

Display sleep is fine. The machine just can't fully power down.

Install Homebrew

Homebrew lets you install software. Open Terminal (Command + Space, type "terminal", Enter).

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

It asks for your password. You won't see letters as you type. That's normal.

Install OpenClaw

OpenClaw runs your AI. It gives it tools, memory, files, and chat access.

brew install openclaw

Then start it:

openclaw gateway install openclaw gateway start

It runs in the background and survives reboots. Close the terminal.

Part 2 — Communication
Connect Telegram

This is how you talk to your AI. Free, works on phone/desktop. Follow OpenClaw's Telegram guide. Once connected, message it. It replies in the same chat.

Also works with Signal, Discord, Slack, iMessage, and others.

Create a Gmail for the AI

Your AI needs its own email to sign up for things. Create a new Gmail on your phone (Google requires text verification, easier on mobile).

Log into it on the AI's machine. Keep it logged in.

Never use your personal Gmail. This is a throwaway identity. If anything goes wrong, you lose nothing important.
Let it create accounts

Tell your AI (via Telegram) to sign up for: GitHub (code), Vercel (hosting), Supabase (databases), or whatever you need.

Optional: give it a Privacy.com virtual card ($20/month limit) for purchases. You type the card number manually when needed.

Part 3 — Knowledge Layer
Install Obsidian

Obsidian lets you see everything your AI writes, beautifully formatted, on all your devices.

brew install --cask obsidian

Open Obsidian. Click "Open folder as vault." Press Cmd+Shift+G and type:

~/.openclaw/workspace
Enable Obsidian Sync

$4/month, end-to-end encrypted. Settings > Core plugins > Sync > on. Log in, create a remote vault.

In Sync settings, add Code to Excluded folders. Keep everything else at defaults.

Do not enable "Sync all file types." Markdown-only is correct. Code repos will overwhelm your sync otherwise.
Keep Obsidian running forever

Save this as ~/Library/LaunchAgents/md.obsidian.keepalive.plist:

<?xml version="1.0" encoding="UTF-8"?> <!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd"> <plist version="1.0"> <dict> <key>Label</key> <string>md.obsidian.keepalive</string> <key>ProgramArguments</key> <array> <string>open</string> <string>-a</string> <string>Obsidian</string> </array> <key>RunAtLoad</key> <true/> <key>StartInterval</key> <integer>300</integer> </dict> </plist>
launchctl load ~/Library/LaunchAgents/md.obsidian.keepalive.plist

Launches on boot, checks every 5 minutes it's still alive.

Connect your other devices

Install Obsidian on your phone/laptop. Create a vault, enable Sync, log in, select the remote vault. Done. No extra config on secondary devices.

Enable semantic search

Makes every note searchable by meaning, not keywords. Runs locally. Zero data sent externally.

openclaw config set agents.defaults.memorySearch.provider local openclaw config set agents.defaults.memorySearch.fallback none

Edit ~/.openclaw/openclaw.json, add your folders under agents.defaults.memorySearch:

"extraPaths": ["Contacts", "Lists", "Memos", "Tasks"]

Restart the gateway. First run downloads a ~313MB model. After that, new files index automatically.

Don't skip this. Without it, the AI only searches its own memory. With it, it searches your entire vault by meaning.
Part 4 — Security
Use a dedicated machine

Never run on your personal computer. The AI gets its own machine, its own user account, no access to your personal files.

Keep the gateway local

The gateway listens on 127.0.0.1 only. Never expose it to the internet. Use Tailscale (free) for remote access.

Write safety rules in SOUL.md

Put these at the top of SOUL.md: no reading outside workspace, no destructive commands, no emails without approval, no spending without approval. It's guidance, not enforcement, but it works.

Use scoped tokens

Every API token: minimum permissions, short expiration, repo-specific. Test: if the AI's identity were compromised, worst case should be a throwaway Gmail and a $20 virtual card.

Set up remote screen access

For the rare times (5%) you need to type for the AI: CAPTCHAs, card fields, browser blocks.

Mac: System Preferences > Sharing > Screen Sharing. Anywhere: Tailscale + VNC.

Part 5 — Go
Share your projects with the AI

GitHub: add as collaborator. Vercel: invite to team. Supabase: add to org. Google Docs: share with its Gmail.

Give it a real task

Don't spend a week configuring. Ask it to do something now. Build a website. Research a topic. Organize your notes. You learn by doing.

Update regularly
openclaw update

Run weekly. New features and security patches ship often.

Pro Tips

Not required. Just things we've found useful.

Pro Tip

Keep folders one level deep. Contacts, Lists, Memos, Tasks. Don't nest three levels deep. Tell your agent how you organize and it will sort files for you.

Pro Tip

Create a Whiteboard.md. Put it at the vault root. "W" sorts near the bottom, easy to find. Use it as a scratchpad: quick thoughts, voice dumps, links to check later.

Pro Tip

All code in Code/. Git repos, cloned projects, skills. Already excluded from Obsidian Sync. Git handles code. Obsidian handles notes. They never overlap.

Known Risks

Prompt injection is unsolved. A malicious email could trick your AI. That's why it gets a throwaway Gmail.

The LLM provider sees your data. Prompts and responses pass through the API. Don't feed it truly secret information.

Skills can be malicious. Only install skills you've reviewed.
