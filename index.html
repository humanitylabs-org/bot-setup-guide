Setup
Get a dedicated machine running with OpenClaw and remote access. After this phase, the AI is alive but you haven't talked to it yet.
Get a dedicated computer

Your AI needs its own machine that never turns off. The best option is a Mac mini — it's small (about the size of a sandwich), silent, and uses very little electricity ($2-3/month). A spare laptop with the lid closed also works. If you don't have spare hardware, you can rent a cloud server (called a VPS) from Hetzner or DigitalOcean for $5-10/month.

This should not be the computer you use every day. Your AI gets its own machine so it can run 24/7 without interrupting your work.

Install Homebrew

First, open Terminal. Terminal is an app on your Mac that lets you type commands directly to the computer. To open it: press the Command key (⌘) and the Space bar at the same time. A search bar called Spotlight appears. Type the word terminal and press Enter. A window with a blinking cursor will open. This is Terminal.

Copy the line below (click it to copy), go to Terminal, paste with Command + V, and press Enter.

copy/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

It will ask for your Mac password (the one you use to log in). When you type it, nothing will appear on screen — no dots, no stars, nothing. That's normal. Just type your password and press Enter.

Installation takes 2-5 minutes. Wait until you see a new blinking cursor.

Prevent the computer from sleeping

Your AI can't work if the computer falls asleep. Open Terminal and paste:

copysudo pmset -a sleep 0

The word sudo means "do this with administrator permission." It asks for your password again (same invisible typing).

Also go to System Settings → Displays → Advanced and turn off "Automatically adjust brightness." If using a laptop, go to System Settings → Battery → Options and turn on "Prevent automatic sleeping on power adapter."

Create a Gmail account for the AI

Your AI needs its own email to sign up for services. Go to Gmail signup on your phone (Google often requires phone verification, easier from a phone). Pick any name — this is a throwaway identity.

Once created, open a browser on the AI's computer, go to gmail.com, and log in with this new account. Leave it logged in.

Never use your personal Gmail. The AI's accounts are separate from yours. If something goes wrong, only a throwaway email is at risk — not your personal life.
Install OpenClaw

Open Terminal and paste:

copycurl -fsSL https://openclaw.ai/install.sh | bash

This downloads and runs the official installer. It installs OpenClaw, registers it as a background service (starts automatically on boot), and starts it immediately. It stays running even if you close Terminal or restart the computer.

If you already have Homebrew, you can also use brew install openclaw followed by openclaw gateway install && openclaw gateway start.

Install Tailscale for remote access

Tailscale (free) creates a private encrypted network between your devices, no matter where they are. You'll use it to access the AI's screen and dashboard from your phone or laptop.

Open Terminal and paste:

copybrew install --cask tailscale

Open Tailscale from Applications. A small icon appears in your menu bar (top right). Click it and sign in with Google, Microsoft, or GitHub. The machine gets a Tailscale IP (like 100.x.x.x) — write it down.

Then install Tailscale on your phone (App Store / Play Store) and laptop (tailscale.com/download). Sign in with the same account on each device.

Enable remote screen access now. On the AI's computer: System Settings → General → Sharing → turn on Screen Sharing. From your Mac: Finder → Go → Connect to Server → type vnc://[Tailscale IP]. From your phone: use a VNC app like Screens (iOS) or bVNC (Android). This lets you see and control the AI's desktop from anywhere — useful for CAPTCHAs, browser issues, and general troubleshooting.
Enable the OpenClaw dashboard

OpenClaw has a web dashboard (Control UI) for checking status, viewing conversations, and managing settings. Enable it:

copyopenclaw config set gateway.controlUI.enabled true openclaw gateway restart

Now from any device on your Tailscale network, open a browser and go to http://[Tailscale IP]:18789. Bookmark it on your phone for quick health checks.

The machine is ready. OpenClaw is running, Tailscale gives you remote access, and the dashboard lets you monitor everything. Next step: connect a messaging app so you can actually talk to your AI.
Chat
Connect a messaging app and start talking to your AI. After this phase, you have a working assistant.
Connect Telegram

Telegram is a free messaging app (like WhatsApp or iMessage). This is how you'll talk to your AI — you send a message, it replies in the same chat, just like texting a friend.

If you don't have Telegram, install it on your phone from the App Store or Play Store. Then follow the OpenClaw Telegram setup guide. It takes about 5 minutes — you create a "bot" through Telegram and paste a token into OpenClaw's settings.

Other options: OpenClaw also works with Signal, Discord, Slack, WhatsApp, and iMessage. Telegram is the easiest to start with.
Send your first message

Open Telegram and send your AI a message: "Hello, are you there?"

It should reply within a few seconds. If it doesn't, check the OpenClaw dashboard (from the Setup phase) to see if there are any errors. Common issues: the Telegram token wasn't saved correctly, or the gateway needs a restart (openclaw gateway restart in Terminal).

Once it replies, congratulations — you have a working AI assistant. Try asking it something real.

Let the AI create its own accounts

Tell the AI to sign up for the services it needs. Send it:

"Sign up for GitHub, Vercel, and Supabase using your Gmail account."

The AI will open a browser, go to each site, and handle registration. You might need to help with CAPTCHAs — the AI will tell you when it's stuck (use the remote screen access you set up in Tailscale).

If you want the AI to make small purchases (like a $11/year domain), create a free virtual card at Privacy.com with a $20/month limit. You type the card number yourself when asked.

Share your existing projects

Invite the AI to your existing stuff so it can collaborate:

GitHub: Repository → Settings → Collaborators → Add the AI's username.

Vercel: Team settings → Members → Invite the AI's Gmail.

Supabase: Organization → Members → Invite the AI's Gmail.

Google Docs: Share button → Enter the AI's Gmail.

You now have a fully working setup. An AI assistant on Telegram that can write code, deploy websites, manage databases, search the web, and research anything. Everything after this makes the experience better — but nothing is required.
Sync
See everything your AI writes on all your devices. Beautifully formatted, end-to-end encrypted, searchable by meaning.
Install Obsidian on the AI's computer

Obsidian is a free app that makes Markdown files (your AI's native format) look beautiful. Open Terminal on the AI's computer and paste:

copybrew install --cask obsidian

Open Obsidian (Command + Space, type "obsidian"). Click "Open folder as vault." Press Command + Shift + G, type ~/.openclaw/workspace, click Open. You'll see all the AI's files: memory logs, notes, configuration, everything.

Enable Obsidian Sync

Obsidian Sync ($4/month) encrypts your files end-to-end and syncs them across all devices. Not even Obsidian can read your content.

In Obsidian: gear icon (⚙️) → Core plugins → toggle on Sync → click Sync in sidebar → log in → Create new remote vault.

In Sync settings, find "Excluded folders" and add Code. Code repos use Git, not Obsidian Sync — mixing them causes problems.

Leave "Sync all file types" off. The default (Markdown only) is correct. Code projects contain thousands of build files that would overwhelm your sync storage.
Keep Obsidian running forever

Obsidian only syncs while open. Make it auto-launch and self-restart. Paste this in Terminal:

copycat > ~/Library/LaunchAgents/md.obsidian.keepalive.plist << 'PLIST' Labelmd.obsidian.keepalive ProgramArguments open-aObsidian RunAtLoad StartInterval300 PLIST launchctl load ~/Library/LaunchAgents/md.obsidian.keepalive.plist

Obsidian will now launch on boot and restart every 5 minutes if closed.

Connect your phone and laptop

Phone: Download Obsidian from the App Store or Play Store.

Laptop: Download from obsidian.md/download.

On each device: open Obsidian → new vault → Settings → Sync → log in with the same account → select the remote vault. Wait a minute for files to download. Everything the AI writes now appears on your phone within seconds.

Enable semantic search

Semantic search lets your AI search all your notes by meaning, not just keywords. Ask "what restaurant did James recommend?" and it finds the answer even if "restaurant" never appears in the note. Runs entirely on your machine — no data sent anywhere.

Open Terminal and paste:

copyopenclaw config set agents.defaults.memorySearch.provider local openclaw config set agents.defaults.memorySearch.fallback none

Tell the AI which folders to search. Open the config:

copyopen ~/.openclaw/openclaw.json

Find memorySearch and add your folders:

"extraPaths": ["Contacts", "Lists", "Memos", "Tasks"]

Save, close, then restart:

copyopenclaw gateway restart

First run downloads a small model (~313MB) and indexes your notes (2-5 min). After that, instant.

This is the most impactful step here. Without it, the AI only searches its own logs. With it, everything in your vault is searchable by meaning.
Security
You've given an AI real tools. These steps keep the risk manageable. Do them at any point.
Dedicated machine, separate accounts

The AI should never run on your personal computer or be logged into your personal accounts. It gets its own machine, Gmail, GitHub, and everything else. If compromised, the blast radius is a throwaway Gmail and free-tier developer accounts — not your personal life.

Gateway stays local

The OpenClaw gateway only listens on 127.0.0.1 (localhost). No one on the internet can reach it. Never expose it publicly. Use Tailscale for remote access — it creates a private tunnel without opening ports.

Write safety rules

Open SOUL.md in your AI's workspace (visible in Obsidian) and add rules like:

"Never send emails, make purchases, or post publicly without asking me first. Never access files outside the workspace. Use trash instead of permanent delete."

Not unbreakable, but significantly reduces unintended actions. Like setting expectations with a new employee.

Use scoped tokens with expiration dates

API tokens should have minimum permissions and expire after 90 days. Ask your AI: "Review your API tokens and flag any with excessive permissions or no expiration."

What's still risky: Prompt injection (malicious content tricking the AI) is unsolved industry-wide — that's why throwaway accounts matter. The LLM provider sees your conversations, so don't share passwords or financial data through the AI. Only install OpenClaw skills from trusted sources.

Pro Tips
Keep your folders one level deep. Contacts, Lists, Memos, Tasks — every note directly in one folder. If you can't find it by filename, rename it. Tell the AI your system and it files things correctly.
Create a Whiteboard.md at the vault root. Sorts to the bottom of your file list. Shared scratchpad: paste links, jot thoughts, dump ideas. Say "check my whiteboard" and the AI knows what you mean.
Put all code in Code/. Every repo, every project, every skill. Excluded from Obsidian Sync — code uses Git, notes use Sync. Tell the AI: "Always clone into Code/."
Give it a real task on day one. "Build me a website." "Research hotels in Kyoto." "Summarize this PDF." You learn by doing, not configuring.
Let the AI finish its own setup. Paste this page's URL into Telegram and say "follow this guide." The AI can read it and complete the Sync and Security steps itself.
Update weekly:
copyopenclaw update

Or just tell the AI: "run openclaw update."
